# Security Policy

## Supported versions

| Version | Supported |
|---------|-----------|
| 0.x (latest) | Yes |

## Reporting a vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities privately via one of these channels:

1. **GitHub private advisory** (preferred): go to [Security → Advisories → New draft advisory](../../security/advisories/new)
2. **Email**: open a GitHub issue mentioning you want to report a vulnerability privately and we will share a contact email

### What to include

- Description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Affected versions
- Any suggested mitigations

### Response timeline

- **Acknowledgement**: within 48 hours
- **Initial assessment**: within 7 days
- **Patch / fix**: depends on severity; critical issues are prioritised

We follow [coordinated disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure): please allow time for a fix to be prepared before public disclosure.
